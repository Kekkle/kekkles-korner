# Update on Level 2 of the 2D Nutrition Game

I've been away from the blog for a bit working on other non-tech ed projects,  but I'm back with updates on Level 2 of the 2D game. TBH, this project has been a bit of a struggle. In my mind's eye, I can see the idea so clearly--how I want the visuals to look, how the game experience feels, what the user "learns" as they go about it--but what is being developed and actually visualized isn't cutting it. 

---

## How It's Been Going

I spent a week working on level 2 which was meant to include all the basic game concepts as level one--jump, collect, shoot--plus a few more features: a new enemy (stress sprites), a new hazard (fog) a ladder feature, and a new game principle of collecting plant and animal proteins and dispensing them in the appropriate "plant" or "animal" basket. 

Being that protein is the structure of all things within the body, I wanted the game visual to have more of a structure feel--bricks, ladders, etc. The first few outputs of the game from Cursor were subpar, but eventually it was sorta close enough to level 1 that I accepted it. But then the game features didn't work smoothly. Climbing ladders was an issue--the character couldn't make it to the top platform. Baskets looked okay, but the drop feature wasn't working. Energy ran out too quickly so the user couldn't complete the game in time. Stress sprites bounced around slowly, adding no challenge. Fog just looked like a lightly transparent area adding nothing to the game. 

I tediously worked on each of those issues, one careful prompt after another. We removed the fog feature, perfected the stress sprite activity (quick shooting action across the screen), fixed the drop button for baskets as well as ladder climbing action. Then the game started to freeze after several seconds of play. Over and over again. After several rounds of Cursor trying to figure out what was up, we decided to remove the ladders. That helped eventually, but I was disappointed because the ladders were a feature I want to continue to show up in the game. I made the decision to remove the baskets as well. Although this is a learning feature that I think is actually useful to the game, it just wasn't working well when playing. There wasn't enough time to complete the game and fill the baskets. Several things needed to be adjusted to make this feature work. 

## What I'm Realizing/Learning

Quite frankly, I am still really struggling with the look of the game. I don't like the graphics. I don't like the background. I don't like how the food items look. I'm okay with the sprites and the mechanics of the game but the visuals make it so that I don't enjoy working on the game or moving forward with it. 

This highlights to me that my original idea of taking a publishing approach was accurate and I should've stuck with it. I should have designed some game screen ideas myself or gathered images of game screens I like and wanted my game to look like, perhaps even designed some of the food items or used an AI generator to create food items in the style I wanted. Once I had my look, feel, tone, color, designs and graphics down, then I should have started on working with Cursor to further develop these front end features, before working on the backend. I've gathered links to several "how-to use Cursor" videos which I'm going through so I can improve not just through trial and error. 

Something I'm learning about myself is that working with AI makes me very eager (impatient?) to see final outcomes. I see the idea in my mind and want to see how Cursor can actualize it, and it's hard to hold myself back and do all the groundwork accurately before getting to the satisfying part of seeing the thing created. And that's on me. I'm going to take a break from the 2D game idea, and work on something smaller, and I'm going to use this next project as an opportunity to put this learning into practice. But more on that later. 

For now, level 2 is complete and playable but not up to standard (and according to my daughter, not fun at all :P ). It's viewable in my projects tab.
